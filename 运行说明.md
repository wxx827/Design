# 基于多设计模式融合的智能AI决策系统

## 项目说明

这是一个完整的Flask + Vue全栈AI系统，融合5种经典设计模式，适合课程设计答辩。

## 技术栈

**后端:**
- Flask 3.0
- Python设计模式实现
- NumPy + Scikit-learn

**前端:**
- Vue.js 3.4
- Vue Router 4
- Pinia状态管理
- Chart.js数据可视化

## 设计模式

1. **策略模式** - 4种AI算法动态切换
2. **工厂模式** - 根据任务类型创建模型
3. **模板方法模式** - 统一AI处理流程
4. **观察者模式** - 实时结果更新
5. **单例模式** - 全局配置管理

## 快速启动

### Windows系统

1. **启动后端**
```bash
start-backend.bat
```
后端运行在: http://localhost:5000

2. **启动前端** (新开一个终端)
```bash
start-frontend.bat
```
前端运行在: http://localhost:3000

### Linux/Mac系统

1. **启动后端**
```bash
chmod +x start-backend.sh
./start-backend.sh
```

2. **启动前端** (新开一个终端)
```bash
chmod +x start-frontend.sh
./start-frontend.sh
```

## 手动启动

### 后端启动

```bash
cd backend
python -m venv venv
# Windows
venv\Scripts\activate
# Linux/Mac
source venv/bin/activate

pip install -r requirements.txt
python app.py
```

### 前端启动

```bash
cd frontend
npm install
npm run dev
```

### 回答示例

**Q: 为什么使用策略模式？**
A: 策略模式允许在运行时动态选择AI算法，4种策略(Baseline/DeepLearning/Attention/Ensemble)独立封装，易于扩展新算法，符合开闭原则。

**Q: 工厂模式的作用？**
A: 根据任务类型(预测/分类/推荐/异常检测)自动创建对应的AI模型实例，解耦对象创建逻辑，提高系统可维护性。

**Q: 如何保证前端实时更新？**
A: 使用观察者模式，当AI结果变化时，Subject通知所有Observer，前端图表自动刷新，实现了发布-订阅机制。

## API接口

- `GET /api/config` - 获取系统配置
- `POST /api/config/task` - 设置当前任务
- `POST /api/config/strategy` - 设置当前策略
- `GET /api/tasks` - 获取任务列表
- `GET /api/strategies` - 获取策略列表
- `POST /api/run` - 运行AI模型
- `GET /api/results` - 获取最新结果
- `POST /api/results/compare` - 对比多个策略
